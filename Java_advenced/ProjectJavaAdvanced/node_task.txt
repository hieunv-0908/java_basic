I. CẤU TRÚC PROJECT CHUẨN
src/
│
├── model/        (Entity - dữ liệu)
├── dao/          (truy vấn DB - JDBC)
│   ├── impl/
│
├── service/      (xử lý nghiệp vụ)
│   ├── impl/
│
├── utils/        (kết nối DB, helper)
├── presentation/ (UI console)
└── Main.java
II. GIẢI THÍCH TỪNG LAYER
1. model (Entity)

👉 Chỉ chứa dữ liệu (class tương ứng bảng SQL)

Ví dụ:

class Order {
    int orderId;
    int userId;
    double totalAmount;
    String status;
}

✔ Không viết SQL ở đây
✔ Không viết logic phức tạp

2. dao (Data Access Object)

👉 Chứa SQL + JDBC

Ví dụ:

interface OrderDAO {
    void insert(Order order);
    Order findById(int id);
    void update(Order order);
}
impl (triển khai)
class OrderDAOImpl implements OrderDAO {

    public void insert(Order order) {
        // viết SQL + JDBC ở đây
    }
}
3. service (Business logic)

👉 Xử lý nghiệp vụ (QUAN TRỌNG NHẤT)

Ví dụ:

class OrderService {

    OrderDAO orderDAO = new OrderDAOImpl();

    public void createOrder(Order order) {
        orderDAO.insert(order);
    }
}
Ví dụ logic nâng cao
public void addItem(int orderId, int serviceId, int quantity) {
    // 1. thêm order_item
    // 2. tính tổng tiền
    // 3. update order
}

👉 Những thứ này không được viết trong DAO

4. presentation (UI Console)

👉 Menu + nhập xuất

public class OrderUI {

    OrderService service = new OrderService();

    public void menu() {
        System.out.println("1. Tạo đơn");
        System.out.println("2. Xem đơn");
    }
}
5. utils

👉 Dùng chung

Ví dụ:

class DBConnection {
    public static Connection getConnection() {
        // kết nối MySQL
    }
}
III. LUỒNG CHẠY CHUẨN
UI → Service → DAO → Database

Ví dụ:

User chọn "Đặt đồ ăn"
→ UI gọi Service
→ Service xử lý logic
→ DAO chạy SQL
→ DB trả dữ liệu
